## 迁移步骤

### 方式一：Git 推送 + 拉取（推荐）

**在当前 VPS 上：**
```bash
cd /opt/blog-src  # 或当前引擎所在目录
git add -A
git commit -m "chore: decouple engine for migration"
git push origin main
```

**在新机器上：**
```bash
# 克隆仓库
git clone <repo-url> ~/quanzhen-night-journal
cd ~/quanzhen-night-journal

# 运行初始化脚本
./setup.sh

# 编辑 .env 配置 API Key 和路径
nano .env

# 手动测试运行
bash scripts/run_night_journal.sh

# 设置 systemd 定时任务（可选）
sudo cp automation/night-journal.service /etc/systemd/system/
sudo cp automation/night-journal.timer /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now night-journal.timer
```

### 方式二：打包 + 手动部署

**在当前 VPS 上：**
```bash
cd /opt/blog-src
tar czf quanzhen-night-journal.tar.gz \
  --exclude='.git' \
  --exclude='content' \
  --exclude='draft_review' \
  --exclude='logs' \
  --exclude='*.json' \
  --exclude='.env' \
  .
```

**在新机器上：**
```bash
mkdir -p ~/quanzhen-night-journal
tar xzf quanzhen-night-journal.tar.gz -C ~/quanzhen-night-journal/
cd ~/quanzhen-night-journal

# 运行初始化脚本
./setup.sh
```

## 注意事项

1. **运行时状态不迁移** - `automation/*.json` 中的运行时状态会在新机器上从模板重新初始化
2. **敏感信息安全** - `.env` 包含 API Key，请勿提交到公开仓库
3. **博客输出目录** - 确保 `BLOG_OUTPUT_DIR` 对应的 Hugo 博客目录可写
4. **Python 依赖** - 新机器需要安装 Python 3，脚本只使用标准库，无需额外依赖
5. **systemd 权限** - 配置 systemd timer 需要 root 权限
