## 3. 后端模块映射

### 3.1 接口层

- `backend/api/auth.py`：登录、会话与初始化状态
- `backend/api/config.py`：配置读写、模型连通性测试、密钥明文解密（reveal）
- `backend/api/personas.py`：人格管理
- `backend/api/memories.py`：记忆列表、检索、更新、提升
- `backend/api/posts.py`：文章列表、详情、编辑、发布
- `backend/api/tasks.py`：手动触发、签发、终止、追踪
- `backend/api/sensory.py`：感知历史与快照查询
- `backend/api/ghost.py`：Ghost 导出、导入、删除、数据库快照
- `backend/api/health.py`：健康检查与系统检查
- `backend/api/audit.py`：审计日志查询与分页

### 3.2 核心引擎

- `ConfigStore`：统一配置读写和加密值处理
- `PersonaEngine`：人格默认值、激活、删除修复
- `MemoryEngine`：记忆创建、检索、文章摘要记忆、最近文章排除
- `SensoryEngine`：感知采样、累计值、增量、速率和历史过滤
- `EventEngine`：事件入库与触发链路承接
- `GenerationOrchestrator`：任务状态机、上下文组装、生成、QA、发布
- `QAEngine`：长度、禁忌词、模板短语、重复检测、完整性检测
- `CostMonitor`：成本记录、日/周/月汇总、预算休眠
- `GhostManager`：`.ghost` 逻辑导出/导入、删除、SQLite 快照备份
- `SiteRuntimeManager`：域名检查、运行时 Hugo/Caddy 配置生成

## 4. 关键运行链路

### 4.1 初始化与启动

1. `backend.main` 启动时执行数据库初始化和自检。
2. `init_database()` 先执行 `doc/database_schema.sql`，再补齐兼容性建表和缺失列。
3. 启动自检会处理以下事项：
   - 恢复异常中断任务状态
   - 确保默认人格存在
   - 确保配置加密密钥存在
   - 可选导入遗留资产

### 4.2 生成与发布

1. 事件来自手动触发、定时任务、Webhook 或目录监控。
2. `GenerationOrchestrator` 拉取人格、记忆、感知和上下文摘要。
3. LLM 生成后进入 `QAEngine`。
4. 若重复、完整性或策略风险过高，则进入失败或人工签发分支。
5. 通过后保存文章、写入 Hugo 内容目录、触发构建并更新状态。

### 4.3 内容一致性

- 文章记忆使用结构化摘要，不回灌整篇正文。
- 最近文章记忆支持“最近 N 篇 + 时间窗”排除。
- 重复检测只比较当前人格下最近已发布文章，并只加载候选文章的向量。
- embedding 不可用时显式标记为 `fallback/manual_review`，由人工处理。

### 4.4 感知与预算

- 感知保留累计值、差值和每秒速率，避免重启或采样间隔波动造成误判。
- 历史查询按 `hours` 真实过滤，不再只看最新 N 条。
- 成本汇总使用明确时间窗，不再依赖字符串前缀匹配日期。

### 4.5 Ghost 与备份

- `.ghost` 导出：逻辑层数据导出，可删除、下载、导入。
- SQLite 快照：原始数据库备份，可创建、列表、下载。
- 大文件上传使用前后端双重限制；后端按块读取并在超限时提前终止。

## 5. 当前部署方式

### 5.1 本地开发

```bash
uv sync --extra dev
uv run uvicorn backend.main:app --reload

cd frontend
npm install
npm run dev
```

### 5.2 容器部署

```bash
cp .env.example .env
docker compose up -d --build
```

默认入口：

- 后台：`http://localhost:5210/admin/`
- API：`http://localhost:5210/api/*`
- 站点：`http://localhost:5210/`

## 6. 验证方式

### 6.1 自动化验证

```bash
uv run pytest backend/tests
cd frontend && npm test
cd frontend && npm run build
```

### 6.2 烟雾验证

```bash
QZ_PASSWORD='<admin-password>' python3 scripts/smoke_test.py
```

该脚本会检查健康接口、后台入口、登录、仪表盘和文章接口。

## 7. 仓库清理约定

### 7.1 应保留

- 源码、配置模板、正式文档
- 当前有效的后后端/前端测试源码
- 正式 Hugo 内容
- 必要的辅助脚本

### 7.2 不应跟踪

- `__pycache__/`
- `.pytest_cache/`
- `frontend/dist/`
- `frontend/node_modules/`
- 运行期数据库、Ghost 导出包、数据库快照
- 运行时生成的 Caddy/Hugo 配置
- 测试报告、中间日志和临时文件

### 7.3 清理原则

- 先区分“缓存/构建产物”和“真实业务数据”。
- 不因为仓库整理而删除测试站正在依赖的数据库和备份。
- 对 Git 已跟踪的临时草稿或失效文档，应直接删除或替换。
- 每次整理结束后必须重新验证并提交，以保证可回滚。
