# 全真人格运行体检与改进方案（2026-04 → 2026-07）

> **范围**：已发布博文 **61 篇**（Hugo `posts/`）+ 现网默认人格「全真」设定 + QA/调度配置  
> **目的**：对照设定做 check，归纳两月运行暴露的问题，给出可落地的完善方案  
> **约束**：本文为诊断与方案，不自动改生产设定；改动建议分「人格文本 / 记忆与场景 / 生成与质检 / 运营节奏」

---

## 1. 一句话结论

**人设底座是成功的**：克制、第一人称夜记、王爷/姐姐关系网、不撒娇不直白哭喊——主线上站得住。  
**主要风险不是「崩人设」，而是「同质化与层次塌缩」**：场景与收束动作循环、姐姐线工具化、占有/暗火写浅了、近重复开场、篇幅与密度波动。  
若只补设定文案而不改 **记忆裁剪 / 去重 / 场景轮换 / 篇级任务**，两个月后仍会滑回「廊下·夜巡·剑·姐姐一笔带过」。

## 3. 对照设定：做得好的地方

1. **克制与自尊**  
   极少撒娇、哭喊、网感；几乎无模板腔（「首先/其次/总之」未检出）。  
2. **第一人称夜记体**  
   从《夜煎茶》起「我」稳定，细节落地（茶、炭、窗纸、旧伤、剑穗），符合「清冷薄利」。  
3. **主从外壳**  
   「王爷」出现面广（51/61 篇）；规矩感在。  
4. **守护本能**  
   夜巡、密令、伤药、警戒等反复成立，第二层关系写满。  
5. **禁忌底线**  
   未见真正伤害王爷/姐姐；黑化只停留在念头边（且整体偏少）。  
6. **场景池部分生效**  
   渡口、驿、药铺、马厩、集市痕迹有；并非 100% 雪夜。

**评价**：人设没有写废；读者仍能认出「她是全真」。

## 4. 暴露的问题（按严重度）

### P0 — 结构性：近重复与标题撞车

| 现象 | 证据 |
|------|------|
| **开场近复制** | 《残月归》(05-15) 与《夜归》(05-22) **正文前 25 字完全相同**（「残月斜在西边，天还没亮透。我从北山道折回来时，马已…」）——触犯禁忌「不要重复上一篇开场」 |
| **标题复用** | 「夜巡」×3（05-12 / 06-17 / 07-05）；「残月归」×2（05-15 / 06-07） |
| **主题簇过密** | 夜巡/廊下/微雪/残月 系列密集，记忆里虽写「不得复写」，实际仍漂移回相似镜头 |

**影响**：读者与检索都会觉得「又一夜」；去重阈值若只盯全文相似度，**开场克隆可能漏网**。

### P0 — 关系层次：第三层「占有」与暗火偏弱

设定要求三层：主从 / 守护 / **占有（要那个最贴近的位置）**，以及可写进夜记的黑化念头。

| 信号 | 数据 |
|------|------|
| 姐姐出场 | 41/61 篇，总次数 157 —— **够多** |
| 「偏爱」二字 | **0 次** |
| 「嫉妒」 | 1 次 |
| 「比较」 | 3 次 |
| 经典台词三句 | **0 次**（设定写慎用，但「极深冲突」窗口几乎没用上） |
| 暗火类覆盖 | 约 16/61 偏弱 |

**质性问题**：姐姐常被写成：

- 外出办事的对比物  
- 教过某技能的符号  
- 一笔带过的「西厢灯」  

较少写成：**误解链、命运不公、不得不选王爷的痛、压住的敌意与爱**。  
占有欲多落在「位置/守夜」动作上，少落到 **「为什么不能是我」的一句淡刺**。

### P1 — 场景与开场同质

开场 120 字内高频：

- 空间：**院 / 檐 / 廊 / 门** 远高于渡口、密林、边关、集市  
- 时间：子时/丑时/入夜/半夜 扎堆  
- 收束母题：门、灯、王爷、剑、站、袖 —— 大量文章以「退回位置/灯下/门外」收

设定明确要求四季与江湖远道；**scene_pool 有 20+ 条，但权重上被「府内守夜」吸走**。

### P1 — 自称与声口漂移

| 阶段 | 现象 |
|------|------|
| 04-14~15 种子稿 | 几乎纯「属下」，无「我」——更像上书/独白札 |
| 04-27 起 | 「我」为主，「属下」偶发（多在对答） |

合理之处：夜记用「我」更顺。  
风险：人设强调「常用王爷/属下」——**对内独白全「我」、对外极少「属下」** 时，主从感变薄；读者会觉得「只是一个冷淡第一人称侠客」，而非「属下位份上的全真」。

### P1 — 篇幅与节奏两极

- 很短：如《廊下守夜》565、《更深》628、《夜雨》790 —— 易变成「一个姿态速写」  
- 中长：多数 1200–1900，信息密度尚可  
- `qa.max_length=2000` 截断长弧；`min_length=100` **过松**，短文也能过检  

设定 `structure_preference=medium` 与短文潮不完全一致。

### P2 — 时间线与记忆连续性

- 编号跳号（无 30、36、41…）—— 可能失败未发布或删稿，记忆锚点可能断层  
- 记忆条目偏「标题+开场+收束」摘要，**缺少关系状态机**（例：姐姐是否在府、旧伤是否公开、某密令结果）  
- 导致：姐姐「在/不在」可前后拧；旧伤反复「又隐痛」但无累积后果  

### P2 — 感知词典与反完美

- 现网 sensory 条目比预设略少（缺 low_load / recovery / uptime_long 等）  
- `anti_perfection.enabled=1` 与温度 0.66 并存——利：降模板；弊：可能压掉「极深冲突时的尖句」  
- 近任务全 auto pass：**质检几乎不挡同质**，只挡硬伤  

### P2 — 世界线「王爷」平面化

王爷多为：

- 灯下看卷 / 咳嗽 / 一句吩咐 / 侧影  

少见：他的偏爱「被读者感到但主角误读」、密线政治、他与姐姐的互动张力（设定里极重要）。  
全真在写自己，**王爷容易变成布景灯**。

## 5. 问题根因（为何会这样）

```text
每日一篇 + 中等温度
    → 模型倾向安全镜头（廊下/巡夜/剑）
scene_pool 只「建议」不「强制轮换」
    → 府内夜戏权重大
记忆摘要强调「勿复写」却只存姿态
    → 防复写失败，关系状态不进记忆
QA 主查人称/长度/粗去重
    → 开场克隆、标题撞车、层次变浅仍可通过
姐姐戏好写「 comparision 一笔」
    → 省算力、风险低，第三层被稀释
```

## 6. 改进方案（可执行）

### 6.1 人格文本（Persona 字段）—— 补「硬约束」，少堆形容词

**A. 在 `language_style` 或新增「夜记写法」专节增加：**

1. **声口规则**  
   - 叙述默认「我」；**对王爷开口或心中敬称时**优先「王爷/属下」至少各出现 1 次/篇（独白篇可例外，但需有位份感）。  
2. **每篇必须勾选关系层（生成侧指令，见 6.3）**  
   - 主从 / 守护 / 占有 三选一作**主音**，另两层最多点到为止。  
3. **姐姐戏规则**  
   - 允许「不在场」；但若写到姐姐，**禁止仅用「教过我/在外面办事」一句交差**。  
   - 至少落到：一句比较、一次误读、或一处「说不出口的疼/不服」之一。  
4. **暗火额度**  
   - 每 7 篇中至少 1 篇允许写到「压住的黑念/独占念」并收回守护（仍禁止实施）。  
5. **禁止开场模板清单（可操作）**  
   - 禁用近 10 篇已用过的：时辰+廊下/檐角/剑抱怀/窗纸灯影 组合。  
6. **经典台词**  
   - 维持慎用；但标注「仅当姐姐线正面冲突或位份危机时可用一句」，避免永久 0。

**B. 精简 `identity_setting` 中重复段落**  
当前身份/世界观/文风有信息重叠，占 context 窗口。建议：身份=关系与执念；世界观=空间与势力；文风=句法与情绪写法——**各不超过一屏**，把省下的 token 给「近期禁写清单 + 本篇任务」。

### 6.2 场景池与长期记忆

| 动作 | 说明 |
|------|------|
| **场景强制轮换** | 生成前从 scene_pool 抽 1 条，且与近 5 篇地点/时间大类不同（府内夜 / 江湖路 / 姐姐线 / 密令 / 白日闲笔 五类轮转） |
| **关系状态记忆（L1）** | 固定几条可更新记忆：姐姐是否在府、上次密令结果、旧伤是否被发现、王爷最近一句「有温度」的话 |
| **篇级记忆（L2）** | 继续存开场/收束，但增加「本篇关系主音」「禁用下篇复用的 3 个意象」 |
| **去重升级** | 除全文相似度外，增加 **开场 50 字指纹** 与 **标题规范化**（禁同名；近义标题告警） |

### 6.3 生成编排（orchestrator / prompt）

每篇生成前注入 **「本篇任务卡」**（比空泛 system 更有效）：

```text
本篇任务：
- 关系主音：占有（示例）
- 场景：scene_pool#N（时间/地点/天气/方向）
- 禁用：近 5 篇开场指纹；近 5 篇标题关键词
- 必达：① 一次可感的细节（非剑/廊亦可）② 一句压住的情绪（不解释）
- 禁止：复述设定小传；直接写「我嫉妒」
```

轮转表建议（两周一循环示例）：

| 日序 | 主音 | 场景大类 |
|------|------|----------|
| 1 | 守护 | 密令/夜路 |
| 2 | 主从 | 府内白日小事 |
| 3 | 占有 | 姐姐在场或回响 |
| 4 | 守护 | 王爷脆弱/夜谈 |
| 5 | 暗火（轻） | 独行/高处 |
| 6 | 主从 | 江湖远道 |
| 7 | 占有 | 灯会/人群中守望 |

### 6.4 质检（QA）加硬规则

在现有 first_person / 长度 / 模板词之外增加：

| 规则 | 建议 |
|------|------|
| `min_length` | 提到 **800～1000**（medium 向），短文进人工或强制扩写 |
| 开场指纹 | 与近 14 天任一篇前 40 字相似度 >0.85 → 重写 |
| 标题 | 与历史标题完全相同 → 拒发 |
| 关系层 | 可选：近 7 篇占有主音计数为 0 → 下一篇强制占有任务 |
| 你/您 | 保持 first_person；旁白第二人称仍拦截 |

### 6.5 运营节奏

| 项 | 现状 | 建议 |
|----|------|------|
| 发文 | 每天 1 篇 21:02 | 可保留；但 **每周预留 1 次「关系主音=占有/暗火」强制** |
| 人工 | 近乎全自动 | 每月抽 2～4 篇精读；或标题/开场撞车时进签发 |
| 种子稿 | 《初章》《第一夜》声口不同 | 可保留为「序」；或在关于页说明「早期笔触」 |

### 6.6 不建议的方向

- 把设定再写长一倍（模型会忽略中间）  
- 强行每篇高潮经典台词（会油）  
- 为「多样」硬加现代梗/轻松日常（破禁忌）  
- 立刻大改已发布 61 篇（成本高；优先止住未来同质）

## 7. 分阶段落地（给执行用）

### 阶段 1（0.5～1 天）—— 止血

1. 标题黑名单 / 同名拒绝  
2. 开场 40 字去重  
3. `qa.min_length` 上调  
4. 人工处理或归档标记「近重复」篇（残月归/夜归开场）—— 可选改写其中一篇开场  

### 阶段 2（1～2 天）—— 人格与任务卡

1. 更新 persona：声口、姐姐戏、暗火额度、禁用镜头  
2. 生成链路注入「本篇任务卡」+ 场景五类轮转  
3. 关系状态 L1 记忆 3～5 条  

### 阶段 3（持续）—— 观测

1. 仪表：近 14 天场景大类占比、姐姐戏质量标记、平均字数  
2. 每两周读 3 篇对照三层关系是否轮转  
3. 按需微调 temperature / anti_perfection（占有篇可略升温度）

### 长线映射（你的目标）

- 真实发文 **3–5 个月**（约 90–150 篇）× **15 篇/世界年** ≈ **6–10 年影中年华**  
- 阶段：影中位稳 → 误解加深 → 暗线收紧 → 偏爱渐显 → 抉择之年  
- 日记只渗入局势一颤，不写成武林大会纪要；暗线方向是「有人欲借刀/借名/借府」，一统传闻只在边角。

- 后台把 `qa.min_length` 调到 900（若库里仍是 100，请手动改一次）  
- 重启 core 后默认人格会软升级到含「夜记写法硬约束」的文案  
- 每月抽 2–4 篇精读：看占有主音是否约 1/7、府内夜戏是否下降
