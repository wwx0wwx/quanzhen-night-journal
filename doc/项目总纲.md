# 全真夜记项目总纲

> 文档版本：2026-04-19
> 状态：当前实现对齐稿

## 1. 项目定位

全真夜记是一个部署在用户自有 VPS 上的数字生命博客系统。它围绕“人格、记忆、感知、事件、生成、发布、审计、迁移”组织，目标不是批量产文，而是让一个具有人格约束的写作主体持续、可解释地存在。

系统当前的完成态是：

- 后端以 FastAPI + SQLite 为核心，负责配置、状态机、生成编排、质检、审计和迁移。
- 前端以 Vue 3 + Vite 管理后台为入口，覆盖初始化、配置、人格、记忆、文章、任务、审计、Ghost 和健康检查。
- 站点发布以 Hugo 为目标，Caddy 负责博客静态页与后台/API 网关。
- 默认部署方式是单机 Docker Compose，优先保证 1C1G/2C2G VPS 可运行。

## 2. 核心目标

- 低门槛部署：通过 `docker compose up -d --build` 在单机完成部署和更新。
- 人格一致创作：生成内容受人格设定、禁忌、表达强度、感知词典和记忆层约束。
- 长期叙事连续性：通过分层记忆和文章摘要记忆维持长期一致性，而不是每次从零开始写。
- 可追溯生成：每篇文章和每个任务都能回溯触发源、上下文、质检结果和最终状态。
- 可控自主性：自动发文受预算、重复检测、内容完整性、人工签发与熔断保护。
- 可迁移与回滚：支持 `.ghost` 逻辑导出/导入，另提供 SQLite 原始快照备份。

## 3. 产品原则

- 人格优先于功能：新增能力不能破坏人格连续性。
- 记忆优先于即时生成：系统价值来自“持续像同一个存在”，不是一次性输出。
- 感知必须人格化：系统采集的是环境线索，不是把监控指标原样贴进文章。
- 自主性必须有护栏：预算、重复检测、完整性校验、人工签发必须可见且可解释。
- 运维动作必须可回滚：导出、备份、部署、配置修改都应留下审计或可恢复介质。

## 4. 当前系统边界

### 4.1 在范围内

- 初始化流程、登录认证、配置管理
- 多人格管理与默认人格切换
- 分层记忆维护、检索、提升与文章记忆归档
- 感知采样、增量/速率计算、历史查询与人格化翻译
- 事件接收：手动、定时、Webhook、目录监控
- 任务编排、Prompt 组装、LLM 调用、QA、重试、人工签发、发布
- Hugo 发布闭环、运行时 Hugo/Caddy 配置生成
- Ghost 导出、预览、导入、删除
- SQLite 原始快照备份、列表、下载
- 健康检查、审计日志、成本统计与休眠控制

### 4.2 不在范围内

- 默认依赖外部向量数据库或分布式基础设施
- 多节点协同发布
- 面向社交平台的多渠道发布编排
- 将运行缓存、构建产物和测试中间结果作为正式仓库内容长期保存

## 5. 当前架构总览

### 5.1 后端

- `backend/api/`：HTTP 接口层，负责认证、参数校验、错误响应和序列化。
- `backend/engine/`：核心领域逻辑，包括记忆、人格、感知、QA、Ghost、任务编排、站点运行时等。
- `backend/models/`：SQLite ORM 模型。
- `backend/scheduler/`：定时任务与采样调度。
- `backend/security/`：JWT、配置加密、Webhook 鉴权。
- `backend/utils/`：时间、JSON、文本处理、审计目录等基础工具。

### 5.2 前端

- `frontend/src/views/`：后台各页面。
- `frontend/src/components/`：通用状态组件、图表和展示组件。
- `frontend/src/stores/`：认证、配置、人格等共享状态。

### 5.3 发布与网关

- `hugo-builder/`：监听构建信号并重建 Hugo 站点。
- `caddy/`：服务博客静态页，同时将 `/admin/` 和 `/api/*` 代理到后台。

## 6. 一致性与防重策略

当前实现已经对齐最近几轮修复，重点如下：

- 文章记忆不再直接把整篇正文原样塞回记忆池，而是转成结构化摘要，避免新稿被旧文全文牵引。
- 文章记忆检索支持“最近 N 篇 + 时间窗”排除，减少刚发布内容对下一篇的强干扰。
- 重复检测优先走 embedding，相似度不足以通过时阻止自动通过。
- 当 embedding 不可用时，系统不会写入伪向量，而是明确标记为 `fallback/manual_review`，要求人工复核。
- 人工签发只允许在 `waiting_human_signoff` 状态执行，避免静默成功。

## 7. 运维与数据边界

- `.ghost` 是逻辑导出，包含人格、记忆、文章、向量和公开配置，不包含容器镜像、主题和证书。
- 数据库快照是 SQLite 原始备份，用于快速回滚运行态，不替代 `.ghost`。
- 仓库只跟踪源码、文档、必要脚本与正式站点内容。
- 构建产物、缓存、运行时配置、测试中间物、数据库、Ghost 包和快照应留在本地或服务器，不进入 Git。

## 8. 文档维护约束

以下内容发生变化时，必须同步更新文档：

- 目录结构或部署方式变化
- Ghost 包能力、备份能力或健康检查口径变化
- 任务状态机、人工签发或重复检测逻辑变化
- 数据库 Schema 与初始化补丁变化
- 仓库清理边界变化
