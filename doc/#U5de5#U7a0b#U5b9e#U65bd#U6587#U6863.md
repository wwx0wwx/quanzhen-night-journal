# 全真夜记 — 工程实施文档

> **用途**：本文档供 AI Coding Agent（如 Claude Code）直接消费，指导从 MVP 到终局形态的全部开发工作。  
> **配套文件**：`项目总纲_v2.md`（产品定义与架构约束）、`database_schema.sql`（建表 SQL）  
> **使用方式**：Claude Code 应先阅读总纲理解"做什么"，再阅读本文档理解"怎么做"。

---

## 1. 工作流程

Claude Code 执行本文档时，必须遵循以下工作流程：

```
Phase 0: 审计现有 MVP → 生成差距报告
Phase 1: 修复 MVP 缺陷 + 对齐数据库 Schema
Phase 2: 完善后端核心模块（人格、记忆、质检、成本）
Phase 3: 实现生成编排引擎（状态机 + Prompt Builder）
Phase 4: 实现感知系统 + 事件引擎
Phase 5: 实现发布器闭环 + Hugo Sidecar
Phase 6: 实现 WebUI 全量页面
Phase 7: 实现高级功能（反完美、Ghost、健康度）
Phase 8: 集成测试 + Docker Compose 调优 + 文档
```

**每个 Phase 的工作原则**：
1. 先读取目标模块对应的总纲章节（§8 模块 N）和本文档的技术规范
2. 检查现有代码中是否已有部分实现
3. 实现/修复/重构，确保与总纲领域模型一致
4. 运行测试验证
5. 进入下一 Phase

## 4. API 规范

### 4.1 通用规约

**基础路径**：`/api`

**统一响应格式**：
```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

**错误码**：
| code | 含义 |
|------|------|
| 0 | 成功 |
| 1001 | 参数校验失败 |
| 1002 | 资源不存在 |
| 1003 | 资源冲突 |
| 2001 | 未认证 |
| 2002 | 认证已过期 |
| 2003 | 权限不足 |
| 3001 | 系统未初始化 |
| 3002 | 预算已耗尽（休眠中） |
| 3003 | 任务锁冲突 |
| 4001 | LLM 调用失败 |
| 4002 | Embedding 调用失败 |
| 4003 | Hugo 编译失败 |
| 4004 | 域名配置失败 |
| 5001 | Ghost 格式错误 |
| 5002 | Ghost 版本不兼容 |

**分页格式**：
```json
{
  "code": 0,
  "data": {
    "items": [],
    "total": 0,
    "page": 1,
    "page_size": 20
  }
}
```

**认证**：
- 登录成功后返回 JWT，存储于 HttpOnly Cookie（`qz_token`）
- 所有 `/api/*` 端点（除 `/api/auth/login`）需携带有效 Cookie
- WebUI 的请求自动携带 Cookie，无需手动管理

## 5. Phase 0 — MVP 审计与修复

Claude Code 拿到项目后的第一个任务是审计现有 MVP。

### 5.1 审计检查清单

逐项检查 MVP 现有代码，标记状态 (✅已实现 / ⚠️部分实现 / ❌缺失)：

**数据库层**：
- [ ] 数据库表是否与 `database_schema.sql` 一致
- [ ] 是否启用 WAL 模式
- [ ] 是否有外键约束
- [ ] 是否有必要的索引
- [ ] updated_at 触发器是否存在
- [ ] 预置配置项是否完整

**认证与安全**：
- [ ] 是否有初始化安全锁（首次登录强制改密码）
- [ ] 密码是否使用 bcrypt hash
- [ ] JWT 是否使用 HttpOnly Cookie
- [ ] API Key 是否加密存储（Fernet）
- [ ] Webhook 是否有鉴权
- [ ] 所有 API 是否需要认证

**后端 API**：
- [ ] 统一响应格式是否符合 §4.1
- [ ] 错误码是否统一
- [ ] 分页是否标准化
- [ ] 是否有请求/响应的 Pydantic Schema 验证

**人格系统**：
- [ ] 人格是否结构化存储（非单段 Prompt）
- [ ] 是否有 sensory_lexicon 字段
- [ ] 是否支持多人格 + 默认人格切换

**记忆系统**：
- [ ] 是否实现 L0/L1/L2/L3 分层
- [ ] 是否有 sqlite-vec 向量表
- [ ] 是否有 Embedding 调用 + 降级策略
- [ ] 是否有衰减机制
- [ ] 是否有回顾机制

**生成引擎**：
- [ ] 是否有 Generation Task 状态机（11 个状态）
- [ ] 状态转移是否合法（参考总纲 §7.5 状态图）
- [ ] Prompt 是否结构化拼装（6 个片段）

- [ ] APScheduler JobStore 是否持久化到 SQLite

### 8.1 状态机 (backend/engine/generation_orchestrator.py)

这是整个系统最核心的文件。状态转移必须严格遵循总纲 §7.5 的状态图。

```python
class GenerationOrchestrator:
    """
    职责: 编排完整的生成链路 (链路 A/B)
    对应总纲: §8 模块 9, §6.5
    """

    VALID_TRANSITIONS = {
        'queued':                ['preparing_context', 'aborted'],
        'preparing_context':     ['generating', 'failed', 'aborted'],
        'generating':            ['qa_checking', 'failed', 'aborted'],
        'qa_checking':           ['ready_to_publish', 'waiting_human_signoff',
                                  'rewrite_pending', 'circuit_open', 'aborted'],
        'rewrite_pending':       ['generating', 'aborted'],
        'waiting_human_signoff': ['ready_to_publish', 'aborted'],
        'ready_to_publish':      ['publishing', 'aborted'],
        'publishing':            ['published', 'failed', 'aborted'],
        # 终态: published, draft_saved, failed, circuit_open, aborted
    }

    async def execute(self, event: Event, persona: Persona = None) -> GenerationTask:
        """执行完整生成链路"""
        task = await self._create_task(event, persona)

        try:
            # 1. Budget Check
            budget = await self.cost_monitor.check_budget()
            if budget.is_hibernating:
                return await self._transition(task, 'failed', error='budget_exhausted')

            # 2. Acquire Task Lock (单活任务)
            if not await self._acquire_lock():
                return await self._transition(task, 'failed', error='task_lock_conflict')

            # 3. Prepare Context
            await self._transition(task, 'preparing_context')
            context = await self.context_builder.build(task, persona)

            # 4. Generate
            await self._transition(task, 'generating')
            content = await self._generate(task, context)

            # 5. QA Check
            await self._transition(task, 'qa_checking')
            qa_result = await self.qa_engine.check(content, persona.id)
            task.qa_result = qa_result.dict()

            if not qa_result.passed:
                if task.retry_count >= task.max_retries:
                    # 熔断
                    await self._save_as_draft(task, content)
                    return await self._transition(task, 'circuit_open')
                else:
                    task.retry_count += 1
                    await self._transition(task, 'rewrite_pending')
                    # 递归重试 (复用同一 Task)
                    return await self._retry_generate(task, context, qa_result)

            if qa_result.risk_level == 'high':
                await self._save_as_draft(task, content)
                return await self._transition(task, 'waiting_human_signoff')

            # 6. Publish
            await self._transition(task, 'ready_to_publish')
            post = await self._create_post(task, content)
            await self._transition(task, 'publishing')
            await self.publisher.publish(post)
            await self._transition(task, 'published')

            # 7. Post-publish: 写入 L2 记忆
            await self.memory_engine.create_from_article(post, persona.id)

            return task

        except Exception as e:
            return await self._transition(task, 'failed', error=str(e))
        finally:
            await self._release_lock()

    async def _transition(self, task, new_status, error=None):
        """状态转移 + AuditLog"""
        old_status = task.status
        if new_status not in self.VALID_TRANSITIONS.get(old_status, []):
            raise InvalidTransition(f"{old_status} -> {new_status}")
        task.status = new_status
        if error:
            task.error_message = error
        if new_status in ('published', 'failed', 'circuit_open', 'aborted', 'draft_saved'):
            task.finished_at = datetime.utcnow()
        await self.db.commit()
        await log_audit(self.db, 'system', 'task.status_change',
                        'task', str(task.id),
                        {'from': old_status, 'to': new_status, 'error': error})
        return task
```

class PromptBuilder:
    """
    职责: 将多来源上下文组装为最终 Prompt
    对应总纲: §9.2
    片段顺序固定:
      1. 系统行为约束
      2. 人格设定
      3. 事件语义
      4. 感知意象
      5. 命中记忆
      6. 输出格式要求
    """

    def build(self, context: GenerationContext) -> tuple[str, dict]:
        """
        返回: (final_prompt, prompt_summary)
        prompt_summary 记录各片段来源 ID 和注入文本的前 100 字，用于 Trace。
        """
        fragments = []
        summary = {}

        # 1. 系统约束
        sys_constraint = self._build_system_constraint()
        fragments.append(sys_constraint)
        summary['system_constraint'] = sys_constraint[:100]

        # 2. 人格设定
        persona_block = self._build_persona_block(context.persona)
        fragments.append(persona_block)
        summary['persona'] = {'id': context.persona.id, 'name': context.persona.name}

        # 3. 事件语义
        if context.event:
            event_block = self._build_event_block(context.event)
            fragments.append(event_block)
            summary['event'] = {'id': context.event.id, 'type': context.event.event_type}

        # 4. 感知意象
        if context.sensory_text:
            fragments.append(f"\n当前身体感知：{context.sensory_text}\n")
            summary['sensory'] = context.sensory_text[:100]

        # 5. 命中记忆
        if context.memory_hits:
            memory_block = self._build_memory_block(context.memory_hits)
            fragments.append(memory_block)
            summary['memories'] = [
                {'id': m.id, 'level': m.level, 'similarity': m.similarity}
                for m in context.memory_hits
            ]

        # 6. 输出格式
        format_block = self._build_format_block(context.persona)
        fragments.append(format_block)

        return "\n\n".join(fragments), summary

## 14. 关键约束提醒

Claude Code 在实施过程中必须始终记住以下约束：

1. **单活任务**：同时最多 1 个 Generation Task 处于非终态
2. **3 次重试硬上限**：熔断后绝不自动恢复
3. **状态转移合法性**：严格按 VALID_TRANSITIONS 字典，非法转移直接抛异常
4. **审计日志覆盖**：所有关键操作必须写 AuditLog
5. **成本记录不可遗漏**：每次模型调用（LLM + Embedding）都写 CostRecord
6. **Budget Check 最前端**：生成链路的第一步就是检查预算
7. **质检不消耗 Token**：四重质检默认纯 Python 实现
8. **感知翻译分两步**：规则标签化 → 人格词典翻译，不直接调 LLM
9. **编译盲区**：Hugo 编译前后 30 分钟的感知数据不进入创作上下文
10. **Prompt Builder 集中化**：所有 Prompt 拼接逻辑集中在一个文件
