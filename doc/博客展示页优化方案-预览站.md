# 博客展示页优化方案（预览站 · 仅中文）

> **原则**：保留极简；**不改动现网** PaperMod 输出；新主题 + **5212** 端口预览；合格后再切换。  
> **备份**：`/projectdev/backups/blog-theme-preview-*`  
> **现网**：`https://iuaa.de` / 容器内 `/srv/hugo`（主题 PaperMod）  
> **预览**：`http://<IP>:5212/`

---

## 1. 现状诊断

| 项 | 现状 |
|----|------|
| 主题 | PaperMod（通用极简，偏工具博客） |
| 气质 | 功能完整，但与「全真夜记」夜札氛围贴合一般 |
| 中文阅读 | 默认西文节奏，行宽/段落可再优化 |
| 首页 | homeInfo + 列表，信息层级一般 |
| 文章页 | 标准 single，可读但少「夜记」辨识度 |
| 风险 | 直接改 PaperMod / 生产 `hugo_public` 会影响读者站 |

---

## 2. 设计方向（极简 + 夜记）

**一句话**：安静、好读、像一本深夜札记，而不是仪表盘。

1. **少即是多**：无重阴影卡片墙、无花哨动效、无社交分享条堆叠  
2. **中文优先**：正文字号约 17–18px、行高 1.8、段落间距舒适、行宽约 40–42rem  
3. **浅色默认**：阅读友好；保留一键深色  
4. **点缀色**：克制墨绿/墨色（与后台 teal 家族呼应，但不照搬 SaaS）  
5. **信息架构不变**：首页 / 归档 / 搜索 / 关于 / 文章详情 / RSS  
6. **只做中文**：`languageCode = zh-cn`，不做多语言 UI  

### 2.1 页面要点

| 页面 | 优化 |
|------|------|
| 首页 | 站名 + 一句简介 + 文章列表（日期 · 标题 · 摘要） |
| 列表/归档 | 按时间清爽罗列，弱化分类噪音 |
| 正文 | 清晰标题区、舒适正文、底栏「上一篇/下一篇」 |
| 搜索 | 保留 JSON 索引客户端搜索（中文关键词） |
| 关于 | 同系排版，少装饰 |

### 2.2 明确不做

- 不改生产 PaperMod 主题文件与现网 `hugo_public`  
- 不上轮播、大封面瀑布流、评论系统  
- 不做英文版展示页

## 3. 技术实现（预览隔离）

```
themes/qz-ink/          # 新主题（夜墨），独立于 PaperMod
hugo/config.preview.toml
scripts/build-blog-preview.sh
caddy/Caddyfile.blog-preview   # :5212
blog-preview/public/           # 预览构建产物（gitignore）
```

| 环境 | 主题 | 输出 | 访问 |
|------|------|------|------|
| 生产 | PaperMod | volume `hugo_public` | 80/443、域名 |
| 预览 | **qz-ink** | `blog-preview/public` | **5212** |

- 预览构建读取**同一套** `hugo_content` 文章，保证内容真实  
- 生产 `qz-hugo` 仍只写 `hugo_public`，互不影响  

### 切换生产（仅你审阅合格后）

1. 将 `hugo/config.toml` 的 `theme` 改为 `qz-ink`（或合并 config.preview 差异）  
2. 重建/触发 hugo 构建，确认 `hugo_public`  
3. 回滚：改回 `PaperMod` + 用备份 `live-public-from-caddy.tar.gz` 或 git

## 5. 预览命令

```bash
# 构建预览静态站
./scripts/build-blog-preview.sh

# 启动/刷新 5212（若未运行）
docker rm -f qz-blog-preview 2>/dev/null
docker run -d --name qz-blog-preview -p 5212:5212 \
  -v "$(pwd)/blog-preview/public:/srv/blog:ro" \
  -v "$(pwd)/caddy/Caddyfile.blog-preview:/etc/caddy/Caddyfile:ro" \
  caddy:2-alpine caddy run --config /etc/caddy/Caddyfile --adapter caddyfile
```
